# Project: Toss Coin Merge (Apps in Toss)
**Date:** 2026-02-04
**Platform:** Apps in Toss (Webview)
**Description:** 토스 인앱 환경을 위한 '수박게임' 스타일의 금융/화폐 머지 퍼즐 게임 개발 프로젝트

---

## Part 1. 블로그 포스팅 초안 (Phase 1 - Ideation)

**제목:** [Apps in Toss] Phase 1. 아이데이션: 토스 유저가 10초 만에 중독될 미니게임 찾기

### 1. Intro: 왜 토스 인앱인가?
토스(Toss) 앱 내에서 실행되는 미니앱, 'Apps in Toss' 개발을 시작했다. 별도의 앱 설치 없이 웹뷰 기반으로 실행된다는 점이 매력적이었고, 무엇보다 내 기술로 토스 유저들에게 소소한 재미를 주고 싶었다.

### 2. Brainstorming: 후보군 탐색
처음엔 기존에 운영하던 `Personalab`의 운세/심리 테스트 데이터를 가져올까 고민했다. 하지만 토스라는 환경 특성상, 텍스트를 읽는 정적인 콘텐츠보다는 **짧은 시간에 몰입할 수 있는 '액션/퍼즐'**이 더 적합하다고 판단했다.

내가 세운 기준은 딱 세 가지다.
1.  **Simple:** 설명서 없이 바로 시작할 수 있어야 한다.
2.  **Theme:** 토스의 아이덴티티인 '금융/돈'과 관련이 있어야 한다.
3.  **Addictive:** 물리적인 재미(타격감, 움직임)가 있어야 한다.

### 3. Final Pick: "동전 합치기 (Money Merge)"
최종적으로 결정한 아이템은 **'동전 합치기'**다.
일명 '수박게임'으로 불리는 머지(Merge) 장르의 규칙을 차용하되, 과일 대신 **동전과 지폐**를 떨어뜨리는 방식이다.

* 10원이 모여 100원이 되고, 5만원권이 모여 골드바가 되는 과정.
* 물리 엔진(Matter.js)을 적용해 동전들이 부딪히고 구르는 예측불허의 재미.

단순히 점수만 오르는 게 아니라, 화면 속의 '자산'이 불어나는 시각적 쾌감을 주는 것이 목표다. 다음 포스팅에서는 이 아이디어를 구현하기 위한 **기술 스택(Tech Stack)** 선정 과정을 다뤄보겠다.

## Part 2. Product Requirements Document (PRD)

### 1. 프로젝트 개요
* **목표:** '수박게임(Suika Game)'의 메커니즘을 차용하여, 동전을 합쳐 더 큰 단위의 화폐를 만드는 캐주얼 퍼즐 게임 개발.
* **타겟 플랫폼:** Apps in Toss (토스 인앱 웹뷰)
* **디자인 컨셉:** TDS (Toss Design System) 스타일의 깔끔하고 직관적인 UI + 쫀득한 물리 효과.
* **핵심 가치:** 단순함, 중독성, 자산이 늘어나는 시각적 쾌감.

### 2. 핵심 기능 및 규칙 (Game Mechanics)
**2.1. 머지(Merge) 규칙**
* 사용자가 화면 터치/드래그로 동전을 떨어뜨릴 위치를 지정하고 손을 떼면 낙하한다.
* **같은 종류**의 화폐 2개가 닿으면 다음 단계의 화폐 1개로 합쳐진다.
* 합쳐질 때 물리적인 반동(Pop effect)과 햅틱 피드백이 발생한다.

**2.2. 화폐 단계 (Currency Progression)**
총 9단계로 구성 (원은 실제 크기 비율을 고려하되 게임적 허용 적용)
1.  **10원** (Smallest)
2.  **50원**
3.  **100원**
4.  **500원**
5.  **1,000원** (지폐 - 원형 토큰화)
6.  **5,000원**
7.  **10,000원**
8.  **50,000원**
9.  **골드바** (Final Stage - 고득점 처리)

**2.3. 게임 오버 조건**
* 화폐가 쌓여서 화면 상단의 **'한계선(Dead Line)'**을 일정 시간(예: 2초) 이상 넘어가면 게임 오버.

### 3. Tech Stack
* **Core:** React (Vite), TypeScript
* **Physics:** `matter-js` (2D 물리 엔진)
* **Styling:** Tailwind CSS (TDS 스타일 구현)
* **Animation:** `framer-motion`
* **State:** Zustand
* **Sound:** Howler.js

### 4. 디자인 가이드 (TDS Style Strategy)
* **Color:** Background `#F2F4F6` / Primary `#3182F6` / Text `#191F28`
* **UI:** 둥근 모서리(`rounded-2xl`), 부드러운 그림자, Pretendard 폰트 사용.
